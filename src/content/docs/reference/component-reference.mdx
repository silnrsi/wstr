## Using the components
To use these components, you need to:
- ensure your Markdown file uses the `.mdx` extension
- import the component(s) you want to use below the frontmatter section but before any other page content, using a declaration such as:
```jsx
import Character from '/src/components/Character.astro';
```
If your component content doesn't show up on the rendered page, check it has the `.mdx` extension and that you've imported the component correctly.

Additional examples of using these components can be found in [Component Coredata Examples](/reference/component-coredata-examples).

## Currently-recommended components

### `Character` component

#### Required import
```jsx
import Character from '/src/components/Character.astro';
```

#### Syntax
```html
<Character usv="XXXX"|char="a" [options="usv,char,name"]/>
```

Note: Either `usv` or `char` must be provided.

|Property|Description|
|---|---|
|usv|The USV of the character to display|
|char|A literal character|
|options|A comma-separated list of options to display. Options are `usv`, `char`, and `name`. The default is all three.|

#### Inline examples

This example uses the default options, where all three elements appear.
```html
<Character usv="0634"/>
```
>Result: <Character usv="0634"/>

This example uses just `usv` and `char` options:
```html
<Character usv="0634" options="usv,char"/>
```
>Result: <Character usv="0634" options="usv,char"/>

#### List example

- <Character usv="0020"/>
- <Character usv="0635"/>

## Components in development

:::note
These components are prototypes for possible future use: their specification is not yet agreed. You are welcome to try them out and give feedback, but be aware that they will likely change.
:::

### `CharacterTable` component

#### Required import
```jsx
import CharacterTable from '/src/components/CharacterTable.astro';
```

#### Syntax
```html
<CharacterTable startUsv="XXXX" endUsv="XXXX" [caption="..."]/>
```

|Property|Description|
|---|---|
|startUsv|The Unicode scalar value of the first character to display|
|endUsv|The Unicode scalar value of the last character to display|
|caption|The caption to display below the table|

#### Examples

##### Source
```html
<CharacterTable startUsv="0627" endUsv="0632" caption="Arabic characters 0627-0632"/> 
```
##### Result
<CharacterTable startUsv="0627" endUsv="0632" caption="Arabic characters 0627-0632"/> 

### `ScriptTable` component

#### Required import
```jsx
import ScriptTable from '/src/components/ScriptTable.astro';
```

#### Syntax
```html
<ScriptTable 
    [filter="key=value,key=value,..."] 
    [order="expression"] 
    [caption="..."]
/>
```
|Property|Description|
|---|---|
|filter|A comma-separated list of key=value pairs to filter the scripts. Possible keys are `script_reordering` and `script_direction`. Possible values for `script_reordering` are `yes` and `no`. Possible values for `script_direction` are `ltr`, `rtl`, `ttb`, and `btt`. Multiple values for a key can be separated by `|`.|
|order|An optional SQL ORDER BY clause to sort the results. Possible fields are `script_name`, `script_code`, `script_type`, `script_family`, `script_direction`, and `script_reordering`.|
|caption|An optional caption to display below the table.|

#### Examples
##### Source
```html
<ScriptTable filter="script_reordering=yes" order="script_family" caption="Scripts with reordering"/>
```
##### Result
<ScriptTable filter="script_reordering=yes" order="script_family" caption="Scripts with reordering"/>
##### Source
```html
<ScriptTable filter="script_type=abjad" order="script_name" caption="Abjad scripts"/>
```
##### Result
<ScriptTable filter="script_type=abjad" order="script_name" caption="Abjad scripts"/>

### `DataTable` component

#### Required import
```jsx
import DataTable from '/src/components/DataTable.astro';
```

#### Syntax
```html
<DataTable 
  table    = "table_name" 
  fields   = "field1[=Label1], field2[=Label2], ..." 
  [filter  = "condition"] 
  [order   = "field1 ASC|DESC, field2 ASC|DESC, ... "] 
  [caption = "Caption text"]
/>
```
|Property|Description|
|---|---|
|table|The name of the table to query. Possible values are `characters`, `scripts`, and `blocks`.|
|fields|A comma-separated list of fields to display. Each field can optionally be followed by `=Label` to specify a custom column header.|
|filter|An optional SQL WHERE clause to filter the results.|
|order|An optional SQL ORDER BY clause to sort the results.|
|caption|An optional caption to display below the table.|

#### Example
##### Source
```html
<DataTable 
  table   = "characters" 
  fields  = "character_usv=USV, character_name=Character Name" 
  filter  = "character_category = 'Lm' AND character_unicodeVersion = '3.0'" 
  order   = "character_usv ASC" 
  caption = "List of Modifier Letters (Lm) added in Unicode 3.0"
/>
```
##### Result
<DataTable 
  table   = "characters" 
  fields  = "character_usv=USV, character_name=Character Name" 
  filter  = "character_category = 'Lm' AND character_unicodeVersion = '3.0'" 
  order   = "character_usv ASC" 
  caption = "List of Modifier Letters (Lm) added in Unicode 3.0"
/>
