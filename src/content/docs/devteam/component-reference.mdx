## Using the components
To use these components, you need to:
- ensure your Markdown file uses the `.mdx` extension
- import the component(s) you want to use below the frontmatter section but before any other page content, using a declaration such as:
```jsx
import Character from '/src/components/Character.astro';
```
If your component content doesn't show up on the rendered page, check it has the `.mdx` extension and that you've imported the component correctly.

Additional examples of using these components can be found in [Component Coredata Examples](/devteam/component-coredata-examples).

## Currently-recommended components

### `ArticlesList` component

Displays a list of articles associated with the given tag.

#### Required import
```jsx
import ArticlesList from '/src/components/ArticlesList.astro';
```

#### Syntax
```html
<ArticlesList [tag="..."] [subset="articlelibonly|topicsonly"] [header="..."] />
```

|Property|Description|
|---|---|
|tag|optional tag to filter the articles listed|
|subset|optional subset of articles to display, either `articlelibonly` or `topicsonly`|
|header|optional header text to display above the list|

Unless `subset` is defined, the component lists contents from both the Article Library and Topic pages.

#### Example

##### Source
```html
<ArticlesList tag="encoding" subset="articlelibonly" header="Encoding Articles" />
```

##### Result

<ArticlesList tag="encoding" subset="articlelibonly" header="Encoding Articles"  />

### `Attribution` component

Display styled attribution elements for an image or article.  This
component allows you specify attribution elements, and formats an 
attribution line based on the license.

#### Required import
```jsx
import Attribution from '/src/components/Attribution.astro';
```

#### Syntax

```html
<Attribution type="Image|Article" 
             license="Public Domain"
             [author="..."]
             [source="..." [sourceurl="..."]]
/>

<Attribution type="Image|Article" 
             license="..."
             [licenseurl="..."] 
             copyholder="..."
             copyyears="..."
             [copyurl="..."]
             [author="..."]
             [source="..." [sourceurl="..."]]
/>
```

|Property|Description|
|---|---|
|type|resource type, can be "Image" or "Article".|
|license| The name of the license for the resource. If it is "Public Domain" then `copyholder`, `copyyears` and `copyurl` properties are ignored. |
|licenseurl| Optional URL that links to the license text, when present it will format the license name as a link. |
|author| Optional name of the author of the resources. |
|copyholder| Copyright holder name. Only required when `license` is not "Public Domain". |
|copyyears| Copyright years, can be a single year, or range of years. Only required when `license` is not "Public Domain". |
|copyurl| Optional URL that links to copyright holder information, when present it will format the `copyholder` as a link. Only required when `license` is not "Public Domain". |
|source| Optional place where the resource came from. |
|sourceurl| Optional URL that links to the website, or information about the source. When present it will format the `source` as a link. |

#### Example

##### Source

```html
<Attribution 
    type="Article" 
    copyholder="John Doe" 
    copyurl="https://example.com/copyholder-info"
    copyyears="2020-2024" 
    license="CC BY-SA 4.0" 
    licenseurl="https://creativecommons.org/licenses/by-sa/4.0/" 
    author="Jane Doe"
    source="Website"
    sourceurl="https://example.com/image-source"
/> 
```

##### Result

<Attribution type="Article" copyholder="John Doe" copyurl="https://example.com/copyholder-info" copyyears="2020-2024" 
  license="CC BY-SA 4.0" licenseurl="https://creativecommons.org/licenses/by-sa/4.0/" author="Jane Doe" 
  source="Website" sourceurl="https://example.com/image-source" /> 

It can also be used with the `<CaptionText />` component:

##### Source

```html
<CaptionText text='Reference: Brase, Jim. "Proposal to encode the Tai Viet script in the UCS", 2007 p. 2' />
<Attribution 
    type="Article" 
    copyholder="John Doe" 
    copyurl="https://example.com/copyholder-info"
    copyyears="2020-2024" 
    license="CC BY-SA 4.0" 
    licenseurl="https://creativecommons.org/licenses/by-sa/4.0/" 
    author="Jane Doe"
    source="Website"
    sourceurl="https://example.com/image-source"
/>
<CaptionText text="This article formerly appeared on ScriptSource." />
```

##### Result

<CaptionText text='Reference: Brase, Jim. "Proposal to encode the Tai Viet script in the UCS", 2007 p. 2' />
<Attribution type="Article" copyholder="John Doe" copyurl="https://example.com/copyholder-info" copyyears="2020-2024" 
  license="CC BY-SA 4.0" licenseurl="https://creativecommons.org/licenses/by-sa/4.0/" author="Jane Doe" 
  source="Website" sourceurl="https://example.com/image-source" /> 
<CaptionText text="This article formerly appeared on ScriptSource." />

### `BibList` component

Displays a list of related sources in a format suitable for a formal bibliography.

#### Required import
```jsx
import BibList from '/src/components/BibList.astro';
```

#### Syntax

```html
<BibList tag="..." [header="..."] [entrytype="non-online|article|etc."]/>
```

|Property|Description|
|---|---|
|tag|optional tag to filter the articles listed; `all` will display all items; if the tag is omitted nothing will be displayed|
|header|optional header text to display above the list|
|entrytype|type of sources to display; valid values are `online`, `article`, `book`, `inbook`, `inproceedings`, `misc`, `non-online`, `all`; defaults to all types|

#### Example

##### Source

```html
<BibList tag='script-cham' header='Bibliography' entrytype='non-online' />
```

##### Result

<BibList tag='script-cham' header='Bibliography' entrytype='non-online' />

### `CaptionText` component

Displays a styled rich text caption line. The caption text can be markdown text,
but be aware the CSS styling will override any markdown styling. 
E.g. If CSS caption styling makes all caption text italicised, then `_` will
have no visible effect.

#### Required import
```jsx
import CaptionText from '/src/components/CaptionText.astro';
```

#### Syntax

```html
<CaptionText text="..."/>
```

|Property|Description|
|---|---|
|text| The text to display as the caption, use basic markdown to add extra styling. |

#### Example

##### Source

```html
<CaptionText text="This is the caption text and _may_ include **basic** [markdown](https://www.markdownguide.org/) styling." />
```

##### Result

<CaptionText text="This is the caption text and _may_ include **basic** [markdown](https://www.markdownguide.org/) styling." />

### `Character` component

#### Required import
```jsx
import Character from '/src/components/Character.astro';
```

#### Syntax
```html
<Character usv="XXXX"|char="a" [options="usv,char,name"]/>
```

Note: Either `usv` or `char` must be provided.

|Property|Description|
|---|---|
|usv|The USV of the character to display|
|char|A literal character|
|options|A comma-separated list of options to display. Options are `usv`, `char`, and `name`. The default is all three.|

#### Inline examples

This example uses the default options, where all three elements appear.
```html
<Character usv="0634"/>
```
>Result: <Character usv="0634"/>

This example uses just `usv` and `char` options:
```html
<Character usv="0634" options="usv,char"/>
```
>Result: <Character usv="0634" options="usv,char"/>

#### List example

- <Character usv="0020"/>
- <Character usv="0636"/>

#### Combining marks
Combining marks are automatically displayed with a dotted circle to indicate their combining nature.
```html
<Character usv="0301"/>
```
>Result: <Character usv="0301"/>

---
#### Directive syntax
```mdx
:usv[XXXX]
:char[a]
:usv[XXXX]{usv char name}
:char[a]{usv char name}
```
These directives are able to work in both `.md` and `.mdx` files.
Please note the `:char` directive is a little more limited, and cannot accept any
whitespace, or certain meta characters without escaping `\[`, `\{`, or wrapping
in inline code quotes.
Astro processes directives after smart quote translation, so `:char[']`
produces `U+2019 ’ RIGHT SINGLE QUOTATION MARK` instead or the expected
`U+0027 ' APOSTROPHE`.  
The following attributes are supported which correspond to the component's `options=` attribute.

|Attribute|Description|
|---|---|
|usv|Display the USV|
|char|Display the character|
|name|Display the Unicode name|

No specified attributes displays all three by default

#### Inline examples

This example uses the default options, where all three elements appear.
```mdx
:usv[0634]
```
>Result: :usv[0634]

This example uses just `usv` and `char` options:
```html
:usv[0635]{usv char}
```
>Result: :usv[0635]{usv char}

This example shows the use of `:char`
```mdx
:char[` `] and :char[@]{name char}
```
>Result: :char[` `] and :char[@]{name char}

#### List example

- :usv[0020]
- :usv[0636]
- :char[$]

#### Table example

|USV|Char|Name|Phoneme|
|---|----|----|-------|
|:usv[0628]{usv}|:usv[0628]{char}|:usv[0628]{name}|/b/|
|:usv[0633]{usv}|:usv[0633]{char}|:usv[0633]{name}|/s/|
|:usv[0634]{usv}|:usv[0634]{char}|:usv[0634]{name}|/:usv[0283]{char}/|
|:usv[0645]{usv}|:usv[0645]{char}|:usv[0645]{name}|/m/|

### `CharacterTable` component

#### Required import
```jsx
import CharacterTable from '/src/components/CharacterTable.astro';
```

#### Syntax
```html
<CharacterTable startUsv="XXXX" endUsv="XXXX" [caption="..."]/>
```

|Property|Description|
|---|---|
|startUsv|The Unicode scalar value of the first character to display|
|endUsv|The Unicode scalar value of the last character to display|
|caption|The caption to display below the table|

#### Examples

##### Source
```html
<CharacterTable
  startUsv = "0627"
  endUsv   = "0632"
  caption  = "Arabic characters 0627-0632"
/>
```
##### Result
<CharacterTable
  startUsv = "0627"
  endUsv   = "0632"
  caption  = "Arabic characters 0627-0632"
/>
